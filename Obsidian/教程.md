# 如何使用 Obsidian 记录笔记（以 Java 学习为例）
[讲解思路](讲解思路.md)
## 1. 介绍

Obsidian 是一款强大的本地知识管理工具，特别适合记录学习笔记。

**什么场景适合用 Obsidian？**
- 学习编程、数学等需要建立知识体系的学科
- 需要频繁引用和关联不同知识点
- 希望完全掌控自己的笔记数据
- 想要长期积累个人知识库

**为什么适合学习 Java？**

学习编程不是孤立地记忆知识点，而是要建立知识之间的联系。比如学习 " 继承 " 时，你需要关联 " 封装 "、" 多态 "、" 接口 " 等概念。Obsidian 的双向链接功能让这种关联变得自然而直观。

## 2. Obsidian 的核心特点

### 2.1 本地存储

所有笔记以 Markdown 文件形式保存在你的电脑上：
- 数据完全由你掌控，不依赖任何云服务
- 可以用任何文本编辑器打开和编辑
- 配合 Git 可以实现版本控制和多设备同步
- 永不过时，10 年后依然可以正常使用

### 2.2 双向链接

这是 Obsidian 最核心的功能，符合人类的思维方式：

```markdown
学习 [[继承]] 时，需要先理解 [[类与对象]]
继承是实现 [[多态]] 的基础
Java 不支持多继承，可以用 [[接口]] 替代
```

当你在 " 继承 " 笔记中链接到 " 多态 " 时，在 " 多态 " 笔记中会自动显示反向链接，告诉你 " 继承 " 引用了它。这样就形成了知识网络，而不是孤立的笔记。

**实际效果：**
- 新学一个知识点时，可以快速找到相关的已学内容
- 通过图谱视图看到整个知识体系的结构
- 发现知识盲区（孤立的笔记说明理解不够深入）

### 2.3 高度可定制

- **主题**：可以安装各种主题，甚至可以用 CSS 自定义样式
- **插件**：社区有上千个插件，可以实现各种功能
- **工作区布局**：可以自由调整编辑器、预览、侧边栏的布局
- **快捷键**：几乎所有操作都可以自定义快捷键

这意味着你可以打造一个完全符合自己习惯的笔记环境。

### 2.4 免费使用

核心功能完全免费，只有云同步（Obsidian Sync）和发布（Obsidian Publish）是付费功能。但我们可以用 Git 实现免费同步，完全够用。

### 3.3 `.obsidian` 文件夹的原理

当你打开一个文件夹作为 Obsidian 仓库时，Obsidian 会自动创建 `.obsidian` 文件夹，里面包含：

```java
.obsidian/
├── app.json          # 应用设置（编辑器、文件等）
├── appearance.json   # 外观设置（主题、字体等）
├── core-plugins.json # 核心插件启用状态
├── community-plugins.json # 社区插件列表
├── hotkeys.json      # 快捷键配置
├── workspace.json    # 工作区布局
├── plugins/          # 已安装的插件
└── themes/           # 已安装的主题
```

**为什么分享模板有用？**

当你分享包含 `.obsidian` 文件夹的仓库时，其他人打开后会自动获得：
- 你配置好的所有设置
- 你安装的所有插件
- 你选择的主题
- 你的工作区布局

这样就省去了大量配置时间，可以直接开始记笔记。

### 3.5 创建第一个笔记

**示例：** `面向对象/继承.md`

```markdown
# 继承（Inheritance）

## 概念

继承是面向对象编程的三大特性之一，允许子类继承父类的属性和方法。

## 基本语法

​```java
public class Animal {
    protected String name;

    public void eat() {
        System.out.println("动物在吃东西");
    }
}

public class Dog extends Animal {
    public void bark() {
        System.out.println("汪汪汪");
    }
}
​```

## 关键点

- 使用 `extends` 关键字实现继承
- Java 只支持单继承
- 子类可以访问父类的 `public` 和 `protected` 成员
- 子类可以重写父类的方法

## 相关概念

- [[封装]]：继承的基础
- [[多态]]：继承的重要应用
- [[接口]]：实现多继承的替代方案
- [[super关键字]]：访问父类成员

## 标签

#面向对象 #继承 #Java基础
```

**核心技巧：**
- 用 `[[笔记名称]]` 创建链接
- 用 `#标签` 分类笔记
- 用代码块展示代码示例

## 4. 进阶技巧

### 4.1 标签的使用

标签用于给笔记分类，方便筛选和查找。

**推荐的标签体系：**

```markdown

**使用方法：**
- 在笔记末尾添加标签

**示例：**
```markdown

### 4.2 同步方案：Obsidian vs 语雀

**语雀的方式：**
- 所有数据存储在云端
- 自动同步，无需配置
- 依赖网络，离线功能受限
- 数据不在自己手中

**Obsidian + Git 的方式：**
- 数据存储在本地，完全掌控
- 通过 Git 实现版本控制和多设备同步
- 可以离线使用
- 需要一些 Git 基础知识

**配置 Git 插件同步：**

1. **安装 Git 插件**
   - 设置 -> 第三方插件 -> 浏览 -> 搜索 "Obsidian Git"
   - 安装并启用

2. **初始化 Git 仓库**
   ```bash
   cd JavaNote
   git init
   git add .
   git commit -m "Initial commit"
   ```

3. **关联远程仓库**
   ```bash
   # 在 GitHub 创建一个私有仓库
   git remote add origin [你的仓库地址]
   git push -u origin main
   ```

4. **配置自动同步**
   - 打开 Git 插件设置
   - 设置自动拉取间隔（如 10 分钟）
   - 设置自动提交和推送

**优势：**
- 完全免费
- 有版本历史，可以回退
- 可以在多台设备间同步
- 数据备份在 GitHub，安全可靠

### 4.3 图片管理

**语雀 vs Obsidian：**

| 功能   | 语雀        | Obsidian   |
| ---- | --------- | ---------- |
| 图片存储 | 自动上传到云端图床 | 默认保存到本地根目录 |
| 粘贴图片 | 自动处理，无需配置 | 需要配置保存路径   |
| 图片管理 | 自动管理      | 需要手动整理     |

**配置 Obsidian 图片保存路径：**

1. 打开设置 -> 文件与链接
2. 找到 " 新附件的默认位置 "
3. 选择 " 指定的附件文件夹 "
4. 输入 `assets` 或 `附件`

这样粘贴图片时会自动保存到 `assets` 文件夹，保持仓库整洁。

**插入图片：**
```markdown
# 方式 1：直接粘贴（推荐）
Ctrl+V 粘贴截图，自动保存到 assets 文件夹

# 方式 2：拖拽
直接拖拽图片文件到编辑器

# 方式 3：手动引用
![图片描述](assets/image.png)
```

## 6. 总结

### 为什么选择 Obsidian？

1. **完全本地化**：数据掌握在自己手中，永不丢失
2. **Markdown 格式**：纯文本，程序员友好
3. **双向链接**：构建知识网络，而不是孤立的笔记
4. **高度可定制**：可以打造完全符合自己习惯的环境

### AI 时代的笔记方式

由于 Obsidian 使用的是纯文本 Markdown 格式，这意味着：

- **AI 友好**：AI 模型（如 Claude、GPT）非常擅长理解和编辑 Markdown 文件
- **自动化潜力**：可以用 AI Agent 自动整理笔记、生成摘要、建立链接
- **未来可期**：随着 AI 技术发展，可以实现更智能的笔记管理

**举例：**
- 让 AI 阅读你的笔记，自动生成知识图谱
- 让 AI 根据你的学习进度，推荐下一步该学什么
- 让 AI 帮你整理错误记录，总结常见问题
- 让 AI 根据你的笔记，自动生成复习题

这是传统笔记软件（如语雀、Notion）难以实现的，因为它们的数据格式不够开放。

### 开始你的 Obsidian 之旅

1. 下载我分享的模板仓库
2. 用 Obsidian 打开
3. 创建第一个笔记
4. 开始建立你的知识网络

记住：**最好的笔记系统是你会持续使用的系统**。不要追求完美，先开始记录，在使用中不断优化。
